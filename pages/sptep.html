1. 시스템 소개

통합 운영 시스템

SPTEP는 철골 생산, 운송, 현장 설치 전 과정을 하나의 플랫폼에서 연결하여 관리합니다. 초기 계획 단계부터 생산 공정, 운송차수, 설치 순서를 함께 고려하여 공정 흐름을 단절 없이 운영하고 전체 프로젝트 효율을 최적화합니다.

주요 목적

  • 생산–운송–설치 공정 통합
  • 운송차수 및 물량 최적화
  • 설치 기준 진척률 관리
  • Firestore 기반 실시간 동기화
  • 권한 기반 협업 운영

핵심 운영 흐름

  • 초기 운송차수계획 → 설치 순서 예측
  • 생산 공정 선제 구성
  • 물량 기반 운송 최적화(25톤 기준)
  • 설치 마킹 기반 실시간 진척률
  • 실시간 공유로 협업 안정화

2. INDEX

Firestore 실시간 공유

SPTEP는 Firestore 기반 실시간 동기화를 중심으로 운영됩니다. 공정표 전체 데이터가 저장되고 변경 즉시 모든 사용자 화면에 동기화되어, 공장(생산)·물류(운송)·현장(설치)이 동일한 최신 데이터를 동시에 확인합니다.

  • 공정표 전체 실시간 동기화
  • 생산·운송·설치 데이터 통합 관리
  • 설치 완료 기준 진척률 자동 계산
  • 프로젝트 참여자 동일 정보 사용
  • 권한 기반 접근 제어
핵심 메시지: 입력 즉시 공유되고, 모두가 같은 공정표를 보고 운영합니다.

3. S — Steel

부재(기둥) 기준 관리

SPTEP는 철골 부재(기둥) 기준으로 전체 공정을 관리합니다. 부재 고유 ID를 기준으로 생산 → 운송 → 설치 전 과정을 연결하여 동일 기준 데이터를 유지합니다.

4. P — Production

멀티 공장 + 대량생산

생산 공정은 생산성 향상을 위해 커팅 공정과 조립 공정을 분리 운영하며, 기둥조립은 Roll Beam / BH Beam 기반 끼움 방식으로 대량생산 체계를 구성합니다.

✏️ 일일 입력 1. 주기둥커팅유석철강(충주공장) 2. 소부재커팅유석철강(오창공장) 3. 주기둥조립유석철강(음성공장) 4. 소부재조립유석철강(음성공장) 5. 현장배송유석철강(음성공장) 6. 현장설치진흥기업

공정 분리 생산 구조

  • 커팅 공장: 기둥커팅 / 소부재커팅
  • 조립 공장: 기둥조립 / 소부재조립
  • 절단 공정 전문화로 병목 최소화
  • 대기시간 감소, 생산 리드타임 단축

기둥조립 대량생산 방식

  • Roll Beam 또는 BH Beam 기반
  • 끼움 방식 조립, 표준화된 반복 생산
  • 제작 시간 단축, 품질 편차 최소화
일일 입력 단계(6) 기둥커팅 · 소부재커팅 · 기둥조립 · 소부재조립 · 현장운송 · 현장설치
상태값 예정 · 진행중 · 완료 · 지연

5. T — Transport

원본/조정 차수 + 25톤 최적화

운송은 차수(Trip) 단위로 계획되며, 설치 순서 및 공정표 구성의 기준 데이터로 활용됩니다. 제작 초기 물량산정을 통해 차량 적재 중량을 사전에 계산하고 25톤 기준으로 운송차수를 최적화합니다.

운송차수계획 (원본)

  • 차수(Trip) 단위로 계획
  • 제작 완료된 부재만 차수 배정
  • 원본 차수표는 기준 집계 데이터로 유지

조정운송차수계획

  • 현장/제작 일정 변경 시 조정 가능
  • 원본 계획을 유지하고 조정 계획을 별도 관리
  • 원본 대비 증감 표시로 차이 즉시 파악
표시 방식 증가: 9개 (+1) · 감소: 9개 (-2) · 변경 없음: 9개
의미 "9개 (+1)" = 조정 후 수량 9개, 원본 대비 1개 증가
기준 고정 표시 기준은 반드시 원본 차수표로 고정 (표시 기준 혼합 금지)
물류 최적화 포인트: 제작 초기 물량산정 → 적재 중량 계산 → 25톤 기준 최적화 → 기존 차수 순서 유지 → 운송차수 감소

* PDF 대신 HTML 표로 즉시 표시. 닫기: X / 바깥 클릭 / ESC

6. E — Erection

설치 마킹 + 진척률 자동 반영

현장 설치 단계에서는 운송된 기둥 부재를 구조물에 접합하고 설치 완료 상태를 시스템에 반영합니다. 설치 완료 기둥은 마킹되어 전체 공정 진척률에 자동 반영됩니다.

설치 단계

  • 반입
  • 설치 진행
  • 설치 완료

구조 접합 방식

  • 볼트 체결 기반 기둥 접합
  • 구조 정렬 후 고정

7. P — Platform

실시간 구독/동기화/이력

SPTEP는 Firestore의 실시간 구독(Realtime Subscribe) 방식으로 동작합니다. 사용자가 공정/운송/설치 데이터를 입력하면 즉시 저장되고, 접속 중인 사용자들은 새로고침 없이 자동으로 갱신된 데이터를 확인합니다.

실시간 동작 방식

  • 입력 즉시 Firestore 저장
  • 사용자 화면 자동 갱신
  • 프로젝트 단위 데이터 분리
  • 변경 이력(누가/언제/무엇을) 기록

실시간 공유 대상

  • 일일 공정표(6단계 + 상태값)
  • 운송차수(원본/조정, 증감 표기 포함)
  • 설치 완료 마킹(진척률 자동 반영)
  • 사용자 권한(Admin/Editor/Viewer)
협업 운영: 모든 입력은 Firestore에 즉시 반영되며 팀원 화면에 실시간 공유됩니다. 설치 완료 마킹은 즉시 진척률에 반영되어 보고서 없이도 현황 파악이 가능합니다. 운송차수 조정 시 원본 대비 증감(예: 9개 (+1))이 실시간 공유되어 커뮤니케이션 오류를 줄입니다.

8. 권한 관리

프로젝트 단위

SPTEP는 프로젝트 단위 권한 체계를 적용합니다. 권한이 없는 사용자는 해당 프로젝트 데이터에 접근할 수 없습니다.

Administrator

  • 전체 공정 데이터 관리
  • 권한 설정 및 사용자 관리
  • 운송 계획 조정

Editor / Viewer

  • Editor: 생산 입력 · 운송 입력 · 설치 업데이트
  • Viewer: 조회 전용 사용자

9. 사용자 매뉴얼

실무 운영 방법

역할별 실무 흐름은 아래와 같습니다. 모든 입력은 실시간 공유되며, 설치 완료 마킹은 진척률에 자동 반영됩니다.

생산 담당자

  • 커팅 입력
  • 조립 상태 업데이트

물류 담당자

  • 운송차수 관리
  • 적재 상태 입력

현장 담당자

  • 반입 등록
  • 설치 완료 마크

관리자

  • 전체 공정 모니터링
  • 지연 공정 관리
운영 핵심 개념 1) 초기 운송차수계획 수립 → 2) 설치 순서 예측 → 3) 생산 공정 선제 구성 → 4) 물량 기반 운송 최적화 → 5) 설치 완료 기반 진척률 관리 → 6) 실시간 협업 운영
